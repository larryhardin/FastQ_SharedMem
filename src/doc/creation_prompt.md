### src/requestMap.js

SharedArrayBuffer-backed in-memory map for tracking job status.

Layout — 109 bytes per slot, up to 1000 slots:
  bytes  0-35:  record UUID       (36 ASCII bytes, generated by randomUUID on add)
  bytes 36-71:  jobId             (36 ASCII bytes, null-padded)
  bytes 72-107: waiting_on_uuid   (36 ASCII bytes, null-padded, optional)
  byte  108:    status            (1 byte, written/read with Atomics)

Status enum (frozen object):
  EMPTY=0, PROCESSING=1, COMPLETED=2, WAITING=3, SKIPPING=4, CANCELLING=5, CANCELED=6, ERROR=7

STATUS_LABEL array (index matches enum):
  [null, 'Processing', 'Completed', 'Waiting', 'Skipping', 'Cancelling', 'Canceled', 'Error']

Internal helpers (not exported):
  _offset(index)                           → index * RECORD_SIZE
  _readField(index, fieldOff)              → reads up to 36 bytes as string, stops at null byte
  _writeField(index, fieldOff, value)      → writes string padded to 36 bytes with null bytes
  _statusCode(status)                      → accepts string label or number; throws on invalid label
  _findEmpty()                             → returns first slot index where status === EMPTY, or -1
  _findSlotByJobId(jobId)                  → returns first non-EMPTY slot index matching jobId, or -1
  _findSlotByUuid(uuid)                    → returns first non-EMPTY slot index matching record UUID, or -1

Exported functions:
  add(jobId, status, waitingOnUuid=null)
    → always inserts a new slot (duplicates allowed for same jobId)
    → validates jobId and waitingOnUuid lengths ≤ 36
    → generates and returns a unique recordUuid string

  read(jobId)
    → returns first matching record as { uuid, jobId, waitingOnUuid: string|null, status: string } or null

  readByUuid(uuid)
    → returns { uuid, jobId, waitingOnUuid: string|null, status: string } or null

  readAllByJobId(jobId, readForStatus=null)
    → returns array of all non-EMPTY records matching jobId
    → if readForStatus is non-null, only include records whose status label matches

  update(uuid, status, waitingOnUuid=null)
    → finds slot by uuid, throws if not found
    → updates status using Atomics.store
    → if waitingOnUuid is non-null, writes it to the waiting_on_uuid field

  remove(jobId)
    → zeroes out the first slot matching jobId; returns boolean

  removeByUuid(uuid)
    → zeroes out the slot matching the record UUID; returns boolean

  check(payload)
    → counts slots matching payload.jobControl.Id; returns number

  list()
    → returns array of all non-EMPTY records as { uuid, jobId, waitingOnUuid, status }

Export: { add, read, readByUuid, update, remove, removeByUuid, check, list, readAllByJobId, STATUS, STATUS_LABEL }

  Four scenarios (evaluated with if blocks, not else-if):

     → Set this._active = false and return undefined (early exit, no handler call).

     → Read currently running tasks: requestMap.readAllByJobId(task.jobControl.Id, 'Processing').
     → Take the first result; call requestMap.update(taskToCancel.uuid, 'Cancelling', null).
     → Call requestMap.update(task.recordUuid, 'Processing', null) again.
